# AI Challenge 2026 Repository

This repository is the workspace for 4 challenge tasks.  
Each task must be implemented in its own folder and submitted with careful validation.

## Challenge Schedule

- 27.04 - Task 1: "vibe coding"
- 04.05 - Task 2: "prototyping with Lovable"
- 11.05 - Task 3: "workflowing with n8n"
- 17.05 - Task 4: "MCPing"

Final deadline: **17.05**

## Repository Structure

```text
.
├── task-1/
├── task-2/
├── task-3/
└── task-4/
```

- Keep each task isolated in its own directory.
- Do not initialize a root application.
- Do not create a root `package.json`.

## Primary Technology Stack (per task project)

- Node.js 24
- React (latest stable)
- React Bootstrap (UI components)
- Font Awesome (icons)

Use this stack as the default unless a specific task announcement says otherwise.

## Task Workflow

1. Read the task announcement fully.
2. Implement only inside `task-X/`.
3. Add artifacts required by the task (links, files, screenshots, docs, etc.).
4. Verify requirements line-by-line before submission.
5. Submit once.

## Submission and Scoring Notes

- Each correct submitted task: **100 points**
- Bonus points apply for top-10 finishers of a task if all requirements are fully correct.
- There is exactly one attempt to qualify for bonus points.
- Any resubmission disqualifies bonus eligibility for that task.

Quality and correctness are more important than speed.

## Developer and AI-Agent Operating Guidelines

### General Rules

- Always start by restating task requirements in checklist form.
- If any requirement is ambiguous, ask clarifying questions before implementation.
- Keep assumptions explicit in `task-X/README.md`.
- Prefer small, verifiable steps over large unverified changes.

### Definition of Done (per task)

- All stated requirements are satisfied.
- Required artifacts are present in the corresponding `task-X/` folder.
- Instructions to run/verify are documented.
- Self-review is completed with a final checklist.

### Recommended Verification Checklist

- Requirement-by-requirement validation complete.
- Build/run steps work on a clean environment.
- No placeholder TODOs left in final submission artifacts.
- Submission links and outputs are correct and reachable.

## Communication with Organizers

If task details are unclear, ask in the related announcement thread and tag the org team.
